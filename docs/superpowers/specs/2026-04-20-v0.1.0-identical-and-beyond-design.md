# v0.1.0 — "Identical and Beyond" — Design

**Milestone:** Arkived v0.1.0 — Azure Blob + ADLS Gen2 foundation
**Status:** Design — awaiting user review
**Date:** 2026-04-20
**Author:** Hamza Abdagić (brainstormed with Claude Opus 4.7)

---

## Summary

v0.1.0 delivers end-to-end Azure Blob and ADLS Gen2 functionality with auth,
CLI, and desktop surfaces wired to a shared core. It replaces the stub IPC in
`app/src-tauri` with real `arkived-core` calls, implements a `Policy` layer
that guards every destructive operation, and preserves Azure Storage Explorer
(ASE) UX familiarity while adding CLI and agent-friendly capabilities ASE
doesn't offer.

The working title is **"Identical and Beyond"**:

- **Identical** — ASE UX parity for everything a migrating ASE user touches:
  dialog titles, button labels, menu items, keyboard shortcuts, auth method
  matrix, feature depth (versions, snapshots, soft-delete, lease, immutability,
  lifecycle).
- **Beyond** — CLI with the full auth matrix plus service principal, managed
  identity, workload identity; policy-gated safety for destructive operations;
  MCP/ACP-ready core (v0.2/v0.4 surfaces); shell completion; JSON/YAML/TSV
  output; per-session policy memory.

## Goals

1. A developer with an Azure storage account can `cargo install arkived`,
   sign in, and complete their daily ASE workflow from CLI or desktop.
2. An ASE user who switches to Arkived doesn't need to re-learn vocabulary or
   keyboard shortcuts.
3. Every destructive operation routes through `Policy::confirm` in
   `arkived-core`. There is no code path that bypasses it.
4. Multiple Entra sign-ins, subscriptions, and storage accounts coexist
   cleanly; the active context is unambiguous in both CLI and UI.
5. No plaintext config file holds credentials or connection metadata.

## Non-goals for v0.1.0

- **Queues, Tables, File Shares surfaces** — deferred to later minor versions.
  Scope exists in `arkived-core` from v0.1.0 but no CLI/UI verbs.
- **MCP server surface** — `arkived-mcp` crate lands in v0.2.0.
- **ACP host** — v0.4.0.
- **Second backend** (S3, GCS, etc.) — v0.5.0.
- **Blob index tags, static website hosting, CORS, change feed, object
  replication, point-in-time restore** — deferred to v0.1.1 (nice-to-have
  per Storage Explorer parity, not critical for daily use).
- **Per-connection persistent policy preferences** — v0.1.0 has session-scope
  memory only; cross-session "always allow X on account Y" is v0.2.0+.
- **Resumable uploads across process restarts** — v0.1.0 is single-process
  resumable only.
- **Tauri plugin extraction** — Tauri commands live in `app/src-tauri`
  directly; factored into a plugin only when a second Tauri app needs them.
- **Telemetry / phone-home** — no default opt-out telemetry in v0.1.0.
- **Pixel-diff UI tests.**

## Key decisions (Q&A summary)

| # | Decision |
|---|---|
| 1 | v0.1.0 scope = **Blob + ADLS Gen2** only |
| 2 | Auth = **ASE UX parity + CLI/agent superset** (service principal, managed identity, workload identity, env/flag) |
| 3 | Credential storage = **OS-native only** (macOS Keychain / Windows Credential Manager / Linux libsecret). **No Azure CLI token-cache sharing.** |
| 4 | Policy triggers = **wide net — 13 destructive actions**; memory = **per-session** |
| 5 | Blob/ADLS depth = **6 must-haves** (versions, snapshots, soft-delete, lease, immutability, lifecycle) |
| 6 | CLI non-interactive auth = **flag > env > `--profile` > keychain > interactive** |
| 7 | Multi-account safety = **current context for reads; explicit `-A` required for writes** |
| 8 | SDK = **official Azure SDK for Rust** (`azure_storage_blobs`, preview `0.21.x`), pinned minor, wrapped behind the internal `StorageBackend` trait |
| 9 | **No config file for connections.** Store-backed SQLite + OS keychain. Config file holds preferences only. |
| 10 | **ASE observable-UX parity** (dialog titles, button labels, shortcuts). CLI/IPC vocabulary stays dev-tool-native. |

```
│  │ + 10    │  │+AzureBlob │  │  + session │ │
│  │ impls   │  │+AzureAdls │  │   memory   │ │
│  └─────────┘  └───────────┘  └────────────┘ │
│       │              │              │        │
│       └──────────────┴──────────────┘        │
│                      │                       │
│              ┌───────┴────────┐              │
│              │     store      │              │
│              │ (SQLite: sign- │              │
│              │  ins, subs,    │              │
│              │   accounts,    │              │
│              │   current ctx) │              │
│              └────────────────┘              │
│                                              │
│                 progress / error / config    │
╰──────────────────────────────────────────────╯
     │                │              │
     ▼                ▼              ▼
   CLI           Tauri desktop    (future:
   (clap)        (src-tauri +     MCP, ACP)
                 React UI)
```

**Dependency rule.** Surfaces depend on `arkived-core`. Core depends on
nothing above it. No lateral dependencies between surfaces.

**Contract.** The internal `StorageBackend` trait stays `pub(crate)`.
`arkived-core`'s public API is Azure-native (`BlobClient`, `ContainerClient`,
`AdlsPathClient`, `AccountClient`, etc.). The trait is an implementation
seam, not a public API.

**Invariant.** Every destructive `StorageBackend` method calls
`policy.confirm(...)` before the SDK call. A compile-time test or clippy
lint enforces it.

## Subsystems

### 1. Auth (`arkived-core/src/auth`)

**`AuthProvider` trait** — single abstraction for every auth method. Returns
an `Arc<dyn TokenCredential>` the Azure SDK consumes.

```rust
#[async_trait]
pub trait AuthProvider: Send + Sync {
    fn kind(&self) -> AuthKind;
    fn display_name(&self) -> &str;
    async fn credential(&self) -> Result<Arc<dyn TokenCredential>>;
    fn supports(&self, resource: ResourceKind) -> bool;
}
```

**Concrete impls — one per ASE method plus CLI/agent additions:**

| Impl | Backing SDK type | Available to |
|---|---|---|
| `EntraInteractive` | `InteractiveBrowserCredential` | Desktop |
| `EntraDeviceCode` | `DeviceCodeCredential` | CLI |
| `AccountKey` | `StorageSharedKeyCredential` | All |
| `ConnectionString` | parses to key+endpoint | All |
| `SasToken` | `AzureSasCredential` | All |
| `Anonymous` | unauthenticated client | All |
| `ServicePrincipal` | `ClientSecretCredential` | CLI / agent |
| `ManagedIdentity` | `ManagedIdentityCredential` | CLI / agent |
| `WorkloadIdentity` | `WorkloadIdentityCredential` | CLI / agent |
| `AzuriteEmulator` | hard-coded dev creds | All |

**Azure environment support.** `AzureEnvironment::{Public, China, USGov, Germany, Custom{url}}`
stored per-connection. One Arkived install can talk to multiple clouds
simultaneously.

**CLI precedence chain** (Q6 decision):

1. Explicit flag (`--sas`, `--connection-string`, `--tenant`, ...)
2. Env var (`ARKIVED_SAS`, `ARKIVED_CONNECTION_STRING`, `ARKIVED_TENANT_ID`, ...)
3. `--profile NAME` → named entry in store + keychain
4. Current-context connection (set via `arkived context use`)
5. Interactive prompt (Entra browser or device code)

### 2. Store (`arkived-core/src/store`)

Single source of truth for connection state. Replaces config-file storage
(ASE-equivalent).

- **Backing:** SQLite file in OS app-data dir (`%APPDATA%\arkived\state.db`
  on Windows, `~/Library/Application Support/arkived/` on macOS,
  `$XDG_DATA_HOME/arkived/` on Linux).
- **Encryption at rest:** database encrypted with a key derived from OS
  keychain entry. If keychain is unavailable, refuse to start with a clear
  error (no fallback to plaintext).
- **Schema:**

  ```sql
  CREATE TABLE sign_in (
      id TEXT PRIMARY KEY,
      display_name TEXT NOT NULL,
      tenant_id TEXT NOT NULL,
      environment TEXT NOT NULL,
      user_principal TEXT NOT NULL,
      added_at TIMESTAMP NOT NULL
  );

  CREATE TABLE subscription (
      id TEXT PRIMARY KEY,
      sign_in_id TEXT NOT NULL REFERENCES sign_in(id) ON DELETE CASCADE,
      name TEXT NOT NULL,
      tenant_id TEXT NOT NULL,
      discovered_at TIMESTAMP NOT NULL
  );

  CREATE TABLE storage_account (
      name TEXT PRIMARY KEY,         -- Azure names are globally unique
      subscription_id TEXT REFERENCES subscription(id) ON DELETE SET NULL,
      kind TEXT NOT NULL,            -- "StorageV2" | "StorageV2 (ADLS Gen2)" | ...
      region TEXT NOT NULL,
      replication TEXT NOT NULL,
      tier TEXT NOT NULL,            -- "Standard" | "Premium"
      hns BOOLEAN NOT NULL,          -- hierarchical namespace (ADLS)
      endpoint TEXT NOT NULL,
      attached_directly BOOLEAN NOT NULL  -- true if attached via SAS/key, not via sign-in
  );

  CREATE TABLE attached_resource (
      -- Resources attached via SAS URL or connection string outside a sign-in
      id TEXT PRIMARY KEY,
      display_name TEXT NOT NULL,
      resource_kind TEXT NOT NULL,   -- "blob-container" | "adls-container" | ...
      endpoint TEXT NOT NULL,
      auth_kind TEXT NOT NULL,
      keychain_ref TEXT NOT NULL     -- pointer to OS keychain secret
  );

  CREATE TABLE context (
      k TEXT PRIMARY KEY,            -- "current_sign_in" | "current_sub" | "current_account"
      v TEXT
  );

  CREATE TABLE policy_memory (
      -- session-scope, cleared on process start
      action_kind TEXT,
      target TEXT,
      allowed_at TIMESTAMP
  );
  ```

- **Secrets are NEVER in the store.** Only keychain references. Each
  `keychain_ref` is a deterministic key (e.g., `arkived:connection:<uuid>`)
  that resolves to the secret via `keyring` crate.

### 3. Backend (`arkived-core/src/backend`)

Internal trait isolating SDK usage.

**`StorageBackend`** (abbreviated; full signature in code):

```rust
#[async_trait]
pub(crate) trait StorageBackend: Send + Sync {
    fn name(&self) -> &'static str;

    // Listing
    async fn list_containers(&self, ctx: &Ctx) -> Result<Page<Container>>;
    fn stream_blobs(&self, ctx: &Ctx, container: &str, prefix: &str, recursive: bool)
        -> impl Stream<Item = Result<BlobEntry>>;

    // Read/write
    fn read_blob(&self, ctx: &Ctx, path: &BlobPath, range: Option<Range>)
        -> impl Stream<Item = Result<Bytes>>;
    async fn write_blob(&self, ctx: &Ctx, path: &BlobPath, body: ByteStream,
                        opts: WriteOpts) -> Result<WriteResult>;

    // Policy-gated (each calls policy.confirm first)
    async fn delete_blob(&self, ctx: &Ctx, path: &BlobPath, opts: DeleteOpts) -> Result<()>;
    async fn set_access_tier(&self, ctx: &Ctx, path: &BlobPath, tier: AccessTier) -> Result<()>;
    async fn set_container_access(&self, ctx: &Ctx, container: &str,
                                   access: PublicAccess) -> Result<()>;
    async fn generate_sas(&self, ctx: &Ctx, scope: SasScope, perms: SasPerms,
                          expiry: DateTime<Utc>) -> Result<String>;
    async fn set_acl(&self, ctx: &Ctx, path: &AdlsPath, acl: Acl) -> Result<()>;

    // Depth (6 must-haves)
    async fn list_versions(&self, ctx: &Ctx, path: &BlobPath) -> Result<Vec<BlobVersion>>;
    async fn create_snapshot(&self, ctx: &Ctx, path: &BlobPath) -> Result<Snapshot>;
    async fn undelete_blob(&self, ctx: &Ctx, path: &BlobPath) -> Result<()>;
    async fn lease(&self, ctx: &Ctx, scope: LeaseScope, op: LeaseOp) -> Result<LeaseState>;
    async fn set_immutability(&self, ctx: &Ctx, path: &BlobPath,
                              policy: ImmutabilityPolicy) -> Result<()>;
    async fn set_lifecycle_rules(&self, ctx: &Ctx, rules: Vec<LifecycleRule>) -> Result<()>;
}
```

**`Ctx`** bundles:
- `Arc<dyn AuthProvider>` — credential factory
- `Arc<dyn Policy>` — confirmation gate
- `Arc<dyn ProgressSink>` — progress event emitter
- `CancellationToken` — cooperative cancel
- `request_id: Uuid` — tracing correlation

Passed to every call so destructive ops have policy, auth, and progress in
one place.

**Implementations:**

- `AzureBlobBackend` — wraps `azure_storage_blobs` in flat-namespace mode.
- `AzureAdlsBackend` — wraps `azure_storage_blobs` in hierarchical mode; owns
  ACL ops (POSIX-style `rwx` on owner/group/other + named entries).
- `AccountDispatcher` — selects backend based on `storage_account.hns`.

**Paging vs streaming.** `Page<T>` for UI (explicit continuation tokens,
pagination controls match the design mockup). `Stream<Item=T>` for CLI
recursive traversal and MCP tool responses (backed by repeated page calls).

**WriteOpts defaults.** 4 MiB block size × 8 parallel uploads. User-tunable
via CLI flags. Resumable within the process; not across process restarts.

**Error taxonomy** (extends existing `arkived-core/src/error.rs`):

```rust
pub enum Error {
    PolicyDenied(String),
    NotFound { resource: String },
    Conflict { detail: String, etag: Option<String> },
    Throttled { retry_after: Duration },
    NetworkTransient(String),
    AuthExpired,
    AuthFailed(String),
    Backend(String),           // catch-all SDK error
    Io(std::io::Error),
    Other(anyhow::Error),
}
```

**Retry policy** — exponential backoff with jitter, 8 attempts max, 30s cap,
only on `Throttled`, `NetworkTransient`. Non-retryable errors propagate
immediately.

### 4. Policy (`arkived-core/src/policy`)

**Trait** — small expansion of the scaffold:

```rust
#[async_trait]
pub trait Policy: Send + Sync {
    async fn confirm(&self, action: &Action, ctx: &ActionContext) -> PolicyDecision;
}

pub enum PolicyDecision {
    Allow,
    AllowAlways { scope: AllowScope },
    Deny(String),
}

pub enum AllowScope {
    ThisAction,                     // one-shot, no memory
    ActionKind,                     // remember "all deletes" this session
    ActionKindOnTarget { target },  // remember "all deletes on stdlnphoenixproddlp" this session
}
```

**The 13 destructive actions (wide net):**

| Verb | Reversible | Billing | Invariant |
|---|---|---|---|
| `delete_blob` | soft-delete window only | ✓ | always gated |
| `delete_container` | no | ✓ | always gated |
| `overwrite_blob` | versions if enabled | ✓ | always gated |
| `rename_blob` | via copy+delete | ✓ | always gated |
| `copy_with_overwrite` | overwrite-side only | ✓ | always gated |
| `purge_soft_deleted` | NO | ✓ | always gated |
| `set_access_tier` | yes (re-tier) | ✓ (early-delete fee) | always gated |
| `set_container_public_access` | yes | — | always gated |
| `set_acl` | yes | — | always gated |
| `set_lifecycle_rules` | yes (edit rule) | ✓ (affects many) | always gated |
| `set_immutability_policy` | depends on mode | — | always gated |
| `generate_sas` | yes (expire/revoke) | — | always gated |
| `break_lease` | yes | — | always gated |

**Per-surface impls:**

- `StdinPolicy` (CLI, TTY) — prints action summary + impact table + command
  preview, reads `y`/`N`/`a` (always for this kind). Honors `--yes` and
  `--yes-always`.
- `NonInteractivePolicy` (CLI, piped/scripted) — `--confirm auto` denies all
  by default; `--confirm yes` blanket-approves (loud-logged).
- `TauriModalPolicy` (desktop) — emits `arkived://policy-request` event,
  awaits `arkived://policy-response`. Three-button modal: Cancel / Approve
  once / Approve for session. UI component already ported in
  `app/src/panels.tsx::ConfirmModal`.
- `McpElicitationPolicy` (v0.2.0, stubbed in v0.1.0)
- `AcpPermissionPolicy` (v0.4.0, stubbed in v0.1.0)

**Invariant enforcement.** A test at compile time (or a custom clippy lint,
stretch goal) walks the AST of `AzureBlobBackend` and asserts every method
on the destructive list calls `policy.confirm` before any SDK call.

### 5. CLI (`arkived-cli`)

**Verb set** — grouped logically. Writes require explicit `-A`, reads use
current context.

```
# auth / context
arkived login [--tenant ID] [--environment azure|china|usgov|custom]
arkived logout [CONNECTION_NAME]
arkived signin add | list | use | rm
arkived subscription list | use
arkived account list | use
arkived attach ...                         # attach via SAS/key outside sign-in
arkived auth add --name NAME --type TYPE ...
arkived auth list | rm

arkived context current                    # print (sign-in, sub, account)

# reads — use current context, -A overrides
arkived ls  [-A ACCOUNT] [CONTAINER[/PATH]] [--recursive] [--format FMT]
arkived cat [-A ACCOUNT] CONTAINER/PATH
arkived stat [-A ACCOUNT] CONTAINER/PATH

# writes — REQUIRE -A even if context is set
arkived cp -A ACCOUNT SRC DST [--overwrite] [--tier HOT|COOL|COLD|ARCHIVE]
arkived rm -A ACCOUNT PATH [--recursive] [--versions] [--soft-delete-keep]
arkived mkdir -A ACCOUNT PATH
arkived mv -A ACCOUNT SRC DST

# capability
arkived sas -A ACCOUNT PATH --perms rwd --expiry 7d [--user-delegation]
arkived acl (get|set) -A ACCOUNT PATH ...
arkived tier -A ACCOUNT PATH --to COOL
arkived lease -A ACCOUNT PATH (--acquire|--renew|--release|--break)
arkived versions -A ACCOUNT PATH
arkived snapshot -A ACCOUNT PATH (--create|--list|--delete)
arkived undelete -A ACCOUNT PATH
arkived immutability -A ACCOUNT PATH (--set|--clear) ...
arkived lifecycle (get|set|delete) -A ACCOUNT ...

# diagnostics
arkived doctor [-A ACCOUNT]
arkived completion bash|zsh|fish|pwsh
```

**Global flags** (clap global):

```
-A, --account NAME
    --connection-string STR
    --sas STR
    --tenant ID
    --profile NAME
    --format json|yaml|table|tsv
    --log-level trace|debug|info|warn|error
    --yes                    # approve this action
    --yes-always             # approve all same-kind this invocation
    --confirm auto|yes       # NonInteractivePolicy
```

**Output format selection** (in order):
1. Explicit `--format` flag
2. Stdout piped → auto JSON (via `is-terminal`)
3. Config file `default_format`
4. `table`

**Exit codes** (sysexits-compatible):

| Code | Meaning |
|---|---|
| 0 | success |
| 1 | generic failure |
| 2 | usage (bad flag) |
| 3 | auth failure |
| 4 | not found |
| 5 | permission denied |
| 6 | policy denied |
| 7 | conflict (ETag / lease) |
| 8 | throttled (retries exhausted) |

**Config file** (preferences only, no credentials or connections):

```toml
default_format = "table"
default_log_level = "info"
default_environment = "azure"
default_confirm = "ask"    # ask | auto | yes
```

Location: `.arkived.toml` (project) > `~/.config/arkived/config.toml` (user).

### 6. Tauri IPC (`app/src-tauri`)

Replaces stub `commands.rs`. Same Rust project; new implementations call
`arkived-core`.

**Command set** (grouped):

```rust
// auth + context
auth_sign_in(environment, tenant) -> Connection
auth_list_connections() -> Vec<Connection>
auth_rm_connection(name)
context_current() -> Option<CurrentContext>     // (sign_in, sub, account)
context_use_sign_in | use_subscription | use_account

// discovery
list_subscriptions(sign_in_id) -> Vec<Subscription>
list_accounts(subscription_id) -> Vec<StorageAccount>
list_containers(account) -> Vec<Container>
list_blobs(account, container, prefix, continuation) -> Page<BlobRow>

// reads
read_blob_head(account, path) -> BlobProperties
read_blob_text(account, path, range) -> String

// writes / policy-gated (every invocation may emit policy-request)
upload(account, dst, src_paths) -> JobId
download(account, src_paths, dst_dir) -> JobId
copy(account, src, dst, overwrite) -> JobId
delete(account, paths) -> JobId
set_tier(account, paths, tier) -> JobId
generate_sas(account, scope, perms, expiry) -> String
set_acl(account, path, acl)
lease(account, scope, op) -> LeaseState
versions(account, path) -> Vec<BlobVersion>
snapshot(account, path, op) -> Snapshot
undelete(account, path)
set_immutability(...)
set_lifecycle(...)

// jobs / progress
job_cancel(job_id)
job_status(job_id) -> Option<JobStatus>

// policy
policy_respond(request_id, decision)
```

**Events (backend → UI):**

```
arkived://policy-request  { action, context, request_id }
arkived://job-progress    { job_id, bytes, total, rate }
arkived://job-complete    { job_id, result }
```

**Policy modal flow:**

1. UI invokes `delete(paths)`.
2. Core calls `policy.confirm(...)`.
3. `TauriModalPolicy` emits `policy-request` event with unique `request_id`,
   awaits channel.
4. UI listens, shows `ConfirmModal`, user picks.
5. UI invokes `policy_respond(request_id, decision)`.
6. Backend channel resolves; core continues or returns `PolicyDenied`.
7. Original `delete()` invocation returns `JobId` or `Err(PolicyDenied)`.

Unique `request_id` per modal eliminates race between concurrent actions.

**Progress streaming.** Uploads/downloads run in tokio tasks. `JobId` returns
immediately. `job-progress` events flow until `job-complete`. UI's
`ActivityBar` subscribes once at mount and routes by `job_id`.

## Testing

**Layers:**

1. **Unit** — pure logic in `arkived-core` (path parsing, SAS URL generation,
   ACL permission math, Policy decision derivation, config precedence).
   No network. Fast.
2. **Azurite integration** — every `StorageBackend` method runs against
   local Azurite in CI. Covers auth (account key, SAS, anonymous), all 13
   policy hooks, paging, streaming, retry-on-throttle (simulated). Default
   in `cargo test --workspace`.
3. **Real Azure E2E** — gated behind `ARKIVED_E2E_CONNECTION_STRING` CI
   secret. Runs only on main-branch pushes. Covers Entra-only paths
   (ADLS ACLs with real RBAC), versions/snapshots/lifecycle (Azurite gaps),
   real throttling. Idempotent with cleanup.
4. **UI Playwright** — smoke tests: sign-in flow, sidebar tree from store,
   Policy modal accept/cancel, upload progress event stream. No pixel diffs.
5. **Policy invariant test** — walks `AzureBlobBackend` AST, asserts every
   destructive method's first call is `policy.confirm`. Prevents
   accidental bypass.

## Observability

- Structured logs via `tracing`. Every SDK call is a span with `request_id`,
  `account`, `operation`, `elapsed_ms`, `result_class`.
- `RUST_LOG` / `ARKIVED_LOG_LEVEL` controls level.
- No telemetry / phone-home in v0.1.0. OpenTelemetry is opt-in and
  user-configured post-1.0 if demand materializes.
- `arkived doctor` self-diagnostic: DNS, clock skew, keychain access,
  Azurite detection, context validity, permission preflight.

## Risks & open questions

- **Azure SDK for Rust is still preview (0.21.x).** Breaking changes across
  versions possible. **Mitigation:** pin minor, wrap behind
  `StorageBackend` so blast radius is one file. If SDK ships 1.0 before
  v0.1.0 GA, adopt. If SDK stalls, option remains to hand-roll a replacement
  with `azure_core` + `reqwest`.
- **MSRV pressure.** Tauri already requires Rust ≥1.77 and is excluded from
  the workspace. Core/CLI may need to bump past 1.75 if `azure_storage_blobs`
  does. Will be decided in Phase A; communicated on next CHANGELOG entry.
- **Encryption-at-rest key derivation.** SQLite store needs a stable key.
  Proposed: random 256-bit key on first launch, stored in OS keychain under
  `arkived:db-key`. Retrieval on every launch via `keyring`. If keychain
  unavailable → refuse to start. No fallback; security is non-negotiable.
- **ASE copy parity drift.** Microsoft can change ASE wording in a patch.
  Our `copy/ase-parity.ts` pins specific strings; drift will require
  periodic maintenance. Acceptable — "identical to ASE *today*" is
  sufficient; we'll track only major shifts.
- **Invariant enforcement.** A clippy lint that checks "every destructive
  method calls policy.confirm first" is non-trivial to write. Fallback:
  runtime test that introspects at compile time with `syn` + `quote` in
  a build.rs. Ugly but guarantees the invariant.

---

## What this spec does NOT decide

Intentionally deferred to the implementation plan:

- Specific `clap` subcommand argument shapes (exact flag names, short
  flags, help text).
- Exact SQLite schema evolution path (migrations library choice, versioning
  strategy).
- Concrete UI copy for every modal and tooltip (handled in Phase D).
- MSRV bump value (determined by the minimum Rust the Azure SDK requires
  at the time Phase A lands).
- Whether the policy-invariant check is compile-time or runtime (pick
  during Phase C).
- Crate-level feature flags (`--no-default-features` behavior, optional
  Entra support, etc.).

These belong in an implementation plan, not a design spec.
